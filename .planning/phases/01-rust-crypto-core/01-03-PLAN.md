    ```rust
    #[uniffi::export]
    pub fn ecdh_and_encrypt(
        our_seed_bytes: Vec<u8>,
        peer_ed25519_public: Vec<u8>,
        contact_fields_json: String,
    ) -> Result<Vec<u8>, PktapError> {
    ```
    1. Validate input sizes: `our_seed_bytes.len() == 32` and `peer_ed25519_public.len() == 32`, else return `PktapError::InvalidKey`.
    2. Convert to fixed arrays: `let seed: [u8; 32] = our_seed_bytes.try_into().map_err(...)` and same for peer key.
    3. Validate plaintext size: `record::validate_plaintext_size(contact_fields_json.as_bytes())?`
    4. Derive encryption key: `let derived = ecdh::ecdh_derive_key(&seed, &peer_pub)?;`
    5. Encrypt: `let record = cipher::encrypt_record(&derived.0, contact_fields_json.as_bytes())?;`
    6. Return `Ok(record)`. The `derived` DerivedKey drops here and zeroes via ZeroizeOnDrop.
    7. Zero the seed array: explicitly zero `seed` with `seed.zeroize()` (import `zeroize::Zeroize` trait) before the function returns. The `our_seed_bytes` Vec should also be dropped.

    Implement `decrypt_and_verify` with `#[uniffi::export]`:
    ```rust
    #[uniffi::export]
    pub fn decrypt_and_verify(
        our_seed_bytes: Vec<u8>,
        peer_ed25519_public: Vec<u8>,
        peer_ed25519_signature: Vec<u8>,
        record_bytes: Vec<u8>,
    ) -> Result<String, PktapError> {
    ```
    1. Validate input sizes: seed 32 bytes, peer key 32 bytes, signature 64 bytes. Return `PktapError::RecordInvalid` for sig/record issues (D-08 coalescing — do NOT return `InvalidKey` for wrong-length signature).
    2. Convert to fixed arrays.
    3. Verify signature first: `signing::verify_signature(&peer_pub, &record_bytes, &sig_bytes).map_err(|_| PktapError::RecordInvalid)?;` — map ANY verification error to `RecordInvalid` per D-08.
    4. Derive encryption key: `let derived = ecdh::ecdh_derive_key(&seed, &peer_pub).map_err(|_| PktapError::RecordInvalid)?;` — map key errors to `RecordInvalid` per D-08 coalescing (do not leak whether key was invalid vs decryption failed).
    5. Decrypt: `let plaintext = cipher::decrypt_record(&derived.0, &record_bytes).map_err(|_| PktapError::RecordInvalid)?;`
    6. Convert to String: `String::from_utf8(plaintext).map_err(|_| PktapError::RecordInvalid)?`
    7. Zero seed bytes.
    8. Return `Ok(json_string)`.

The FFI caller must not be able to distinguish decryption failure from verification failure. Internal Rust logging (if any) may log the specific cause, but the returned error is always `RecordInvalid`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Kotlin (FFI caller) -> ecdh_and_encrypt | Untrusted seed bytes, peer public key, and JSON payload cross FFI |
| Kotlin (FFI caller) -> decrypt_and_verify | Untrusted seed, peer key, signature, and record bytes cross FFI |
| Network (via DHT, future) -> decrypt_and_verify | Record bytes from DHT may be forged or tampered |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Information Disclosure | ffi::decrypt_and_verify | mitigate | D-08 error coalescing: ALL internal errors map to single `PktapError::RecordInvalid` across FFI — prevents decryption/verification oracle |
| T-01-10 | Tampering | ffi::ecdh_and_encrypt input | mitigate | Validate input byte lengths (seed=32, peer_key=32) before any crypto operation; reject with typed error |
| T-01-11 | Elevation of Privilege | ffi::ecdh_and_encrypt seed lifetime | mitigate | Seed bytes `.zeroize()` after use; DerivedKey drops with ZeroizeOnDrop before function returns |
| T-01-12 | Spoofing | ffi::decrypt_and_verify signature | mitigate | Ed25519 signature verified BEFORE decryption; forged records rejected before key derivation |
| T-01-13 | Denial of Service | ffi::ecdh_and_encrypt payload size | mitigate | `validate_plaintext_size` rejects payloads > 750 bytes with `PktapError::RecordTooLarge` before encryption |
</threat_model>
