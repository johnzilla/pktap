    1. Call `keys::validate_ed25519_public_key(peer_ed25519_pub)` to validate the peer key.
    2. Call `keys::ed25519_pub_to_x25519_pub(&verified_key)` to get the peer's X25519 public key.
    3. Call `keys::seed_to_x25519_scalar(our_seed)` to derive our X25519 private key from the HKDF seed.
    4. Create `StaticSecret::from(scalar.0)` and `PublicKey::from(peer_x25519_pub.to_bytes())`.
    5. Call `our_secret.diffie_hellman(&peer_public)` to get the shared secret.
    6. Check `shared.as_bytes() == &[0u8; 32]` — if true, return `Err(PktapError::InvalidKey)` per RESEARCH.md Pitfall 3.
    7. Create `Hkdf::<Sha256>::new(None, shared.as_bytes())`.
    8. Call `hk.expand(b"pktap-v1", &mut okm)` where `okm` is `[0u8; 32]`.
    9. Return `Ok(DerivedKey(okm))`.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | keys::validate_ed25519_public_key | mitigate | `VerifyingKey::from_bytes()` rejects invalid Edwards points; returns typed `PktapError::InvalidKey` |
| T-01-02 | Tampering | ecdh::ecdh_derive_key | mitigate | Check `shared.as_bytes() == &[0u8; 32]` after every `diffie_hellman()` call; reject low-order points with `PktapError::InvalidKey` |
| T-01-03 | Information Disclosure | ecdh::DerivedKey, keys::X25519ScalarBytes | mitigate | `#[derive(ZeroizeOnDrop)]` on all secret newtypes; secrets zeroed on scope exit |
| T-01-04 | Elevation of Privilege | Secret material lifetime | mitigate | No raw `[u8; 32]` for secrets — all wrapped in ZeroizeOnDrop newtypes; `SharedSecret` consumed inline, never stored |
</threat_model>
