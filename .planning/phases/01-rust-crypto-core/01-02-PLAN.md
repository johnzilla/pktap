    Implement `pktap-core/src/cipher.rs`:

    Import: `chacha20poly1305::{XChaCha20Poly1305, XNonce, KeyInit, aead::{Aead, AeadCore}}`, `rand_core::OsRng`.

    Define constant: `pub(crate) const RECORD_VERSION: u8 = 0x01;`
    Define constant: `pub(crate) const NONCE_LEN: usize = 24;`
    Define constant: `pub(crate) const VERSION_LEN: usize = 1;`
    Define constant: `pub(crate) const TAG_LEN: usize = 16;`
    Define constant: `pub(crate) const FIXED_OVERHEAD: usize = VERSION_LEN + NONCE_LEN + TAG_LEN;` (= 41 bytes)

    Implement `pub(crate) fn encrypt_record(key: &[u8; 32], plaintext: &[u8]) -> Result<Vec<u8>, PktapError>`:
    1. Create `XChaCha20Poly1305::new(key.into())`.
    2. Generate nonce: `XChaCha20Poly1305::generate_nonce(&mut OsRng)` — produces 24-byte `XNonce`.
    3. Encrypt: `cipher.encrypt(&nonce, plaintext).map_err(|_| PktapError::SerializationFailed)`.
    4. Construct D-06 byte blob: allocate `Vec::with_capacity(1 + 24 + ciphertext.len())`, push `0x01` version byte, extend with nonce bytes, extend with ciphertext (which already includes the 16-byte Poly1305 tag appended by the AEAD).
    5. Return the Vec.

    Implement `pub(crate) fn decrypt_record(key: &[u8; 32], record: &[u8]) -> Result<Vec<u8>, PktapError>`:
    1. Check `record.len() >= FIXED_OVERHEAD` (minimum: 1 version + 24 nonce + 16 tag), else return `PktapError::RecordInvalid`.
    2. Check `record[0] == RECORD_VERSION`, else return `PktapError::RecordInvalid`.
    3. Extract nonce: `XNonce::from_slice(&record[1..25])`.
    4. Extract ciphertext+tag: `&record[25..]`.
    5. Create `XChaCha20Poly1305::new(key.into())`.
    6. Decrypt: `cipher.decrypt(nonce, ciphertext_and_tag).map_err(|_| PktapError::RecordInvalid)`.
    7. Return plaintext bytes.

    Add `#[cfg(test)] mod tests` with the behavior tests above. For the IETF KAT, use the test vector from draft-irtf-cfrg-xchacha-03 section 2.2 if available, or construct a deterministic round-trip test with a fixed key and nonce (using `XNonce::from_slice` instead of random generation for the test).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FFI caller -> cipher module | Untrusted record bytes may be tampered; AEAD authentication prevents forgery |
| FFI caller -> signing module | Untrusted signatures from peer; Ed25519 verify rejects invalid signatures |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Information Disclosure | cipher::encrypt_record | mitigate | Use `OsRng` for nonce generation (24-byte XNonce); never reuse nonces; 192-bit nonce space makes random collision negligible |
| T-01-06 | Tampering | cipher::decrypt_record | mitigate | AEAD authentication tag (Poly1305) rejects any modification to ciphertext; returns `PktapError::RecordInvalid` |
| T-01-07 | Tampering | signing::verify_signature | mitigate | `VerifyingKey::verify()` uses constant-time comparison; rejects invalid signatures with `PktapError::RecordInvalid` |
| T-01-08 | Repudiation | record::shared_record_name | accept | DNS name derived from sorted public keys is deterministic; no non-repudiation guarantee needed at this layer (DHT records are signed separately by Pkarr) |
</threat_model>
