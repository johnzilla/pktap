# Phase 04 Plan 01: Keystore Infrastructure and Rust FFI Exports Summary

**One-liner:** AES-256-GCM Keystore key with StrongBox/TEE fallback encrypts 32-byte seed at rest; derive_public_key and derive_mnemonic_from_seed BIP-39 FFI exports added to Rust core with ByteArray zeroing in Kotlin bridge.

## What Was Built

### Task 1: Rust FFI exports + PktapBridge wrappers

- Added `bip39 = "2.2.2"` to `pktap-core/Cargo.toml`
- Created `pktap-core/src/mnemonic.rs` — `mnemonic_from_entropy(&[u8; 32])` using first 16 bytes as entropy (128-bit → 12 BIP-39 words)
- Added `mod mnemonic` to `lib.rs`
- Added two `#[uniffi::export]` functions to `ffi.rs`:
  - `derive_mnemonic_from_seed(seed_bytes: Vec<u8>) -> Result<String, PktapError>` — validates 32-byte length, calls `mnemonic_from_entropy`, zeroes local seed array
  - `derive_public_key(seed_bytes: Vec<u8>) -> Result<Vec<u8>, PktapError>` — validates 32-byte length, derives Ed25519 `SigningKey`, returns `verifying_key().to_bytes()`, zeroes local seed array
- Wrote 7 Rust unit tests covering: 12-word count, determinism, wrong-length error, 32-byte return, dalek match — all pass
- Regenerated UniFFI Kotlin bindings (new functions appear in generated `pktap_core.kt`)
- Updated `PktapBridge.kt` with `deriveMnemonicFromSeed` and `derivePublicKey` wrappers, both zeroing `seedBytes` in `finally` blocks (D-05)

**Verification:** `cargo test -p pktap-core test_derive` — 9 tests pass (7 new + 2 pre-existing derive_shared_record_name tests).

### Task 2: KeystoreManager + SeedRepository + instrumented tests

- `KeystoreManager.kt` — `object` with:
  - `generateOrGetKey(alias)`: checks KeyStore for existing key; on API 28+ tries StrongBox, catches `StrongBoxUnavailableException` and falls back to TEE; on API 26-27 skips StrongBox entirely
  - `encrypt(plaintext, key)`: AES/GCM/NoPadding, auto-generated IV, returns `"iv_b64:ciphertext_b64"`
  - `decrypt(stored, key)`: splits on `:`, reconstructs IV + ciphertext, returns plaintext
  - `SEED_KEY_ALIAS = "pktap_aes_key"`
- `SeedRepository.kt` — `class` taking `Context` with:
  - `hasSeed()`, `isMnemonicAcknowledged()`, `setMnemonicAcknowledged()`
  - `generateAndStoreSeed(): ByteArray` — SecureRandom 32 bytes, Keystore encrypt, stores in prefs, resets mnemonic flag, runs on `Dispatchers.IO`
  - `decryptSeed(): ByteArray` — reads prefs, Keystore decrypt, catches `KeyPermanentlyInvalidatedException` (clears prefs, throws `SeedInvalidatedException`), runs on `Dispatchers.IO`
  - No seed field on class (T-04-06)
- 4 `KeystoreManagerTest` + 5 `SeedRepositoryTest` instrumented tests — all 9 pass on `pktap-test` AVD
- `app/build.gradle.kts`: `testInstrumentationRunner`, `coroutines.android`, `test.runner`, `test.ext.junit` deps added
