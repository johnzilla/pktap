<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Mnemonic display -> user's eyes | 12 words shown on screen — screenshots, screen recording, shoulder surfing |
| ViewModel memory | Mnemonic words held in StateFlow during display |
| SharedPreferences flag | mnemonic_acknowledged boolean — if tampered, could skip mnemonic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Information Disclosure | Mnemonic captured via screenshot | mitigate | FLAG_SECURE on mnemonic screen window via DisposableEffect. Prevents screenshots and recent apps thumbnails. |
| T-04-09 | Information Disclosure | Mnemonic words in logcat | mitigate | Never log word list. ViewModel logs operation names only ("Mnemonic derived"), never word values. |
| T-04-10 | Information Disclosure | Mnemonic words surviving in ViewModel after navigation | accept | Words are in StateFlow which is GC-eligible after ViewModel is cleared. Risk is minimal — words are in user's memory anyway. Clearing StateFlow on navigate is possible but unnecessary. |
| T-04-11 | Tampering | mnemonic_acknowledged flag set without viewing words | accept | SharedPreferences is app-private. Rooted device could tamper, but user would only be skipping their own backup — no security impact to others. |
| T-04-12 | Spoofing | Fake seed injected via SharedPreferences | accept | SharedPreferences is MODE_PRIVATE. Rooted device risk accepted — same as all Android local storage. Keystore key is non-extractable regardless. |
| T-04-13 | Information Disclosure | Seed in MnemonicViewModel during derivation | mitigate | Seed decrypted in IO coroutine, passed to PktapBridge (which zeros it internally), local copy zeroed in finally block. Seed never assigned to a ViewModel property. |
</threat_model>
