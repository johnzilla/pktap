<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Kotlin -> Rust FFI | Seed bytes cross from JVM heap to Rust memory via UniFFI |
| Android Keystore -> JVM | Decrypted seed bytes leave hardware security module into JVM heap |
| SharedPreferences file | Encrypted seed at rest on device filesystem |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | Seed in JVM memory after FFI call | mitigate | Zero ByteArray in finally block immediately after PktapBridge call (D-05). Verified by existing zeroing tests in Phase 3. |
| T-04-02 | Information Disclosure | Seed logged via crash reporter or logcat | mitigate | Never log seed bytes; KeystoreManager and SeedRepository log only operation names ("Key generated via TEE"), never byte values. No toString() on ByteArray secrets. |
| T-04-03 | Tampering | AES-GCM IV reuse breaks confidentiality | mitigate | Never provide IV to Cipher.init(ENCRYPT_MODE) — let Keystore auto-generate random IV on each encrypt. Store IV alongside ciphertext. |
| T-04-04 | Denial of Service | KeyPermanentlyInvalidatedException after factory reset | mitigate | Catch exception in SeedRepository.decryptSeed(), clear prefs, signal re-setup needed (Pitfall 5). |
| T-04-05 | Elevation of Privilege | StrongBox downgrade to TEE | accept | Both StrongBox and TEE provide non-extractable keys. TEE is the minimum security bar and is acceptable for this threat model. D-02 documents this as intentional. |
| T-04-06 | Information Disclosure | Seed stored as class field surviving GC cycles | mitigate | SeedRepository returns seed as local ByteArray only. No class-level seed property anywhere. Code review checklist: "no seed field on any class" (Pitfall 3). |
| T-04-07 | Tampering | API 26-27 crash from setIsStrongBoxBacked | mitigate | Guard with `Build.VERSION.SDK_INT >= Build.VERSION_CODES.P` (API 28). On API 26-27, skip StrongBox entirely. |
</threat_model>
